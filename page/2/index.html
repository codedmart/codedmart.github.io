MongoDB And MongoMapper Greatness

I was so excited to get a blog up and running and I haven’t even posted on it yet. So here goes nothing. I have been reading about document-oriented databases for a little while, but never really dug into it too far. It just so happens that Railscasts did a segment on MongoDB and MongoMapper so this was the perfect time to look into it a little more. I must say that John Nunemaker and anyone else that has worked on MongoMapper is doing a great job because it was pretty easy for me to convert our very own zyphmartin.com site over from PostgreSQL to MongoDB. Keep in mind that I do not consider myself a SQL guru to begin with so I am quite excited and impressed with MongoDB and MongoMapper.

Setting up the connection was very straight forward just as the Railscast shows:

# initializers/mongo_config.rb
# Set Logger
MongoMapper.connection = Mongo::Connection.new('localhost', 27017, :logger => Rails.logger)
 
# Database
MongoMapper.database = "zm-#{Rails.env}"

# Authentication
MongoMapper.database.authenticate('user', 'password')

That is it to get up and running and you technically don’t need to set the logger or authentication to get a base minimum setup going. I ended up setting the logger originally because I was using a modified version of Paperclip but I ended up switching to Carrierwave since it works out of the box with MongoMapper. Another reason I choose Carrierwave was because I am able to save to file as I did with Paperclip or I can save to Mongo’s GridFS.

I also opted out of Thinking Sphinx which is what I was using for the search on the site but I thought that is was a little heavy for what we needed so I went to a simple mongo query. I will be adding more blog posts about MongoMapper down the road. Most everything else worked fine without much change.
